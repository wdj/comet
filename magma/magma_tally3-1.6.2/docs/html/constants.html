Constants

MAGMA_tally3 defines a few constant parameters, such as Magma_tally3Trans, Magma_tally3NoTrans, that are equivalent of CBLAS and LAPACK parameters.

The naming and numbering of these parameters follow that of CBLAS from Netlib and the C Interface to LAPACK from Netlib, and PLASMA.

MAGMA_tally3 includes functions, lapack_xyz_const(), which take MAGMA_tally3's integer constants and return LAPACK's string constants, where xyz is a MAGMA_tally3 type such as uplo, trans, etc. From the standpoint of LAPACK, only the first letter of each string is significant. Nevertheless, the functions return meaningful strings, such as "No transpose", "Transpose", "Upper", "Lower", etc. Similarly, there are functions to go from MAGMA_tally3's integer constants to CBLAS, OpenCL's clBLAS, and CUDA's cuBLAS integer constants.

There are also functions, magma_tally3_xyz_const(), to go in the opposite direction, from LAPACK's string constants to MAGMA_tally3's integer constants.

The most common constants are those defined for BLAS routines:

  • enum { Magma_tally3NoTrans, Magma_tally3Trans, Magma_tally3ConjTrans } magma_tally3_order_t

    Whether a matrix is not transposed, transposed, or conjugate-transposed. For a real matrix, Trans and ConjTrans have the same meaning.

  • enum { Magma_tally3Lower, Magma_tally3Upper, Magma_tally3Full } magma_tally3_uplo_t

    Whether the lower or upper triangle of a matrix is given, or the full matrix.

  • enum { Magma_tally3Left, Magma_tally3Right } magma_tally3_side_t

    Whether the matrix is on the left or right.

  • enum { Magma_tally3Unit, Magma_tally3NonUnit } magma_tally3_diag_t

    Whether the diagonal is assumed to be unit (all ones) or not.

Additional constants for specific routines are defined in the documentation for the routines.

Because MAGMA_tally3, CBLAS, LAPACK, CUBLAS, and clBlas use potentially different constants, converters between them are provided.

These convert LAPACK constants to MAGMA_tally3 constants. Note that the meaning of LAPACK constants depends on the context: 'N' can mean False, NoTrans, NonUnit, NoVec, etc. Here, curly braces { } group similar constants.
